# FAFULI 工具箱用户操作手册

本手册用于说明如何使用本地页面 `index.html` 中的工具：

- 保全方案编号匹配工具
- 商城订单按供应商拆分
- 超龄名单匹配工具

所有数据都在本机浏览器内处理，不需要网络连接，也不会上传到服务器。

## 1. 打开工具箱

1. 打开 `FAFULI 工具箱` 文件夹。
2. 双击 `index.html`。
3. 浏览器打开后，即可看到左侧目录和右侧工具内容区。

![工具箱整体页面](docs-assets/toolbox-home.png)

左侧目录目前包含三个分组：

- `保险运营`
  - `保全方案编号匹配工具`
- `平台运营`
  - `商城订单按供应商拆分`
- `客户服务`
  - `超龄名单匹配工具`

点击左侧目录名称，可以在右侧切换不同工具。

## 2. 保全方案编号匹配工具

该工具用于上传保全主表和映射表，自动匹配保险公司方案编号，并按保险公司拆分导出 Excel 文件。

![保全方案编号匹配工具](docs-assets/insurance-tool.png)

### 2.1 准备文件

需要上传 4 个 `.xlsx` 文件：

| 文件 | 必要字段 | 用途 |
| --- | --- | --- |
| 主表：保全名单 | `公司ERP客编`、`方案编号`、`产品编号`、`保险公司`、`医保属地`、`与主险人关系` | 作为需要处理的保全数据 |
| 映射表：保险方案映射关系 | `公司ERP客编`、`方案编号`、`产品编号`、`与主险人关系`、`医保属性`、`保险公司`、`保险公司方案编号` | 通过多字段精确匹配保险公司方案编号 |
| 映射表：医保属性映射关系 | `医保属地`、`医保属性`、`人保健康属性` | 生成导出表中的医保属性 |
| 映射表：特殊保险公司映射关系 | `特殊保险公司名称`、`保险公司名称` 或 `保险公司` 任一字段 | 判断哪些保险公司需要使用人保健康属性 |

说明：

- 工具只读取每个 Excel 的第一个工作表。
- 不要上传 Excel 临时锁文件，例如文件名以 `~$` 开头的文件。
- 如果表头缺失，页面会提示错误，并阻止导出。

### 2.2 上传文件

1. 在 `1. 上传文件` 区域，依次上传 4 个文件。
2. 文件上传成功后，文件名会以蓝色显示。
3. 如果上传错文件，点击对应上传框内的 `删除` 按钮后重新上传。

### 2.3 查看校验结果

文件读取成功后，页面会按保险公司分别展示校验结果：

| 字段 | 含义 |
| --- | --- |
| 保险公司名称 | 主表中的保险公司 |
| 数据行数 | 该保险公司的数据行数量 |
| 匹配成功 | 成功匹配到保险公司方案编号的数量 |
| 未匹配 | 未匹配到保险公司方案编号的数量 |

如果存在未匹配，页面会展示未匹配明细，包括保险公司名称、方案编号和出现行数。

### 2.4 医保属性规则

导出表中会新增 `医保属性` 字段，规则如下：

1. 如果主表 `与主险人关系` 为 `子女`，医保属性直接填写 `有医保`。
2. 如果保险公司为 `中国人民健康保险股份有限公司上海分公司`，使用医保映射表中的 `人保健康属性`。
3. 如果保险公司命中特殊保险公司映射表，也使用医保映射表中的 `人保健康属性`。
4. 其他保险公司使用医保映射表中的 `医保属性`。
5. 如果医保属地没有匹配到映射关系，医保属性填写 `未匹配`。

导出字段名称始终为 `医保属性`，不会额外导出 `人保健康属性` 字段。

### 2.5 保险公司方案编号匹配规则

工具会先按医保属性规则生成每行的 `医保属性`，再使用以下 6 个字段精确匹配保险方案映射关系：

`公司ERP客编`、`方案编号`、`产品编号`、`与主险人关系`、`医保属性`、`保险公司`

映射表中上述 6 个匹配字段不允许为空。如果 6 个匹配字段完全重复但 `保险公司方案编号` 不同，页面会提示冲突并停止校验，需要修正映射表后重新上传。

### 2.6 特殊字段说明

以下 4 家保险公司的导出文件会在表格最后增加专属字段，字段值默认留空：

| 保险公司 | 追加字段 |
| --- | --- |
| 百年人寿保险股份有限公司浙江分公司 | 被保人人序号、保险计划、份数、职业代码、职业类别、国籍 |
| 亚太财产保险有限公司深圳分公司 | 保险责任、赔付比例、免赔额、是否含一级医院、一年期标准保费、险种、保单号、批次、批改信息、批改保费、生效止期、投保项目、既往症名称 |
| 阳光财产保险股份有限公司苏州中心支公司 | 年龄、职业代码、职业类别、批改标志、保司方案代码 |
| 中国人民健康保险股份有限公司上海分公司 | 状态、年龄、国籍、生僻字标识、职业代码、职业类别、岗位、保险计划 |

其中，阳光财产保险股份有限公司苏州中心支公司和中国人民健康保险股份有限公司上海分公司的 `年龄` 字段，会根据主表中的 `被保险人出生日期` 自动计算。

### 2.7 导出文件

1. 确认校验结果无误。
2. 点击 `导出 ZIP`。
3. 浏览器会下载一个 ZIP 压缩包。
4. ZIP 中每个保险公司对应一个 Excel 文件。

导出规则：

- 每个保险公司一个 `.xlsx` 文件。
- 文件名格式为：`保险公司名称_日期.xlsx`。
- 空保险公司名称会导出为 `未填写保险公司_日期.xlsx`。
- 文件名中的非法字符 `/ \ : * ? " < > |` 会自动替换为 `_`。
- 未匹配的 `保险公司方案编号` 会填写 `未匹配`，并在 Excel 中高亮。
- 以下字段会排在特殊字段之后、表格最后：方案特别约定、客服、客户名称、是否高风险人群、重疾名称、客服维护时间、集团户名称、保全提取时间、是否理赔高风险。

## 3. 商城订单按供应商拆分

该工具用于上传商城订单主表，按 `供应商` 拆分成多个 Excel 文件，并打包成 ZIP 导出。

![商城订单按供应商拆分](docs-assets/mall-order-tool.png)

### 3.1 准备文件

需要上传 1 个 `.xlsx` 文件：

| 文件 | 必要字段 | 用途 |
| --- | --- | --- |
| 商城订单主表 | `供应商` | 按供应商拆分订单 |

建议主表中包含 `结算总价` 字段。页面会按供应商汇总结算总价，并从高到低排序展示。

### 3.2 上传文件

1. 在上传区上传商城订单主表。
2. 上传成功后，文件名会以蓝色显示。
3. 如果上传错文件，点击 `删除` 后重新上传。

### 3.3 查看供应商预览

上传成功后，页面会展示供应商汇总预览：

| 字段 | 含义 |
| --- | --- |
| 供应商 | 主表中的供应商名称 |
| 订单行数 | 该供应商的订单数量 |
| 结算总价 | 该供应商的结算总价合计 |
| 文件名 | 导出时生成的 Excel 文件名 |

预览结果会按照 `结算总价` 从高到低排序。

### 3.4 导出文件

1. 确认供应商预览无误。
2. 点击导出按钮。
3. 浏览器会下载 ZIP 压缩包。
4. ZIP 中每个供应商对应一个 Excel 文件。

导出规则：

- 每个供应商一个 `.xlsx` 文件。
- 文件名格式为：`供应商_日期.xlsx`。
- ZIP 文件名格式为：`商城订单_按供应商拆分_日期.zip`。
- 导出表保留订单主表字段，但会删除以下字段：所属企业、企业编号、集团户名称、会员名称、会员手机号、会员证件号、积分售价、优惠类型、优惠总金额、订单金额、超级积分金额、商城积分金额、现金支付金额、第三方支付方式、支付类型、支付订单号、第三方支付流水。

## 4. 超龄名单匹配工具

该工具用于上传人员清单主表，根据 `与主险人关系` 和 `被保人生日` 筛选达到年龄条件的员工、子女名单，并导出新 Excel。

### 4.1 准备文件

需要上传 1 个 `.xlsx` 文件：

| 文件 | 必要字段 | 用途 |
| --- | --- | --- |
| 主表：人员清单 | `与主险人关系`、`被保人生日` | 根据身份和被保人生日计算年龄并筛选人员 |

建议主表中包含 `被保险人姓名`、`被保险人证件号`、`主险人姓名` 字段，页面预览会优先展示这些信息。

### 4.2 匹配规则

工具按浏览器当天日期计算整周岁，并导出以下人员：

1. `与主险人关系` 为 `员工`、`本人` 或 `主险人`，且年龄大于等于 65 周岁。
2. `与主险人关系` 为 `子女`，且年龄大于等于 18 周岁。

如果 `被保人生日` 无法识别，工具会尝试从 `被保险人证件号` 中读取出生日期。

### 4.3 查看匹配预览

上传成功后，页面会展示符合条件的人员明细：

| 字段 | 含义 |
| --- | --- |
| 与主险人关系 | 员工、子女等关系 |
| 被保险人姓名 | 人员姓名 |
| 被保人生日 | 主表中的被保人生日 |
| 年龄 | 按当天日期计算的整周岁 |
| 主险人姓名 | 主表中的主险人姓名 |
| 匹配规则 | 当前人员命中的保留规则 |

页面顶部会汇总匹配人数、员工人数、子女人数和未保留人数。

### 4.4 导出文件

1. 确认匹配预览无误。
2. 点击 `导出 Excel`。
3. 浏览器会下载筛选后的人员新表。

导出规则：

- 文件名格式为：`超龄名单匹配结果_日期.xlsx`。
- 导出表保留原主表字段。
- 导出表最后追加 `年龄` 和 `匹配规则` 两列。

## 5. 常见问题

### 5.1 为什么上传后提示缺少表头？

请检查上传的 Excel 第一个工作表是否包含页面要求的字段名称。工具按表头名称识别字段，如果字段缺失或名称不一致，会阻止导出。

### 5.2 为什么有方案编号未匹配？

说明主表数据没有通过 `公司ERP客编`、`方案编号`、`产品编号`、`与主险人关系`、`医保属性`、`保险公司` 这 6 个字段匹配到保险方案映射关系。请检查主表、医保属性映射结果和保险方案映射表中的字段值是否一致。

### 5.3 为什么医保属性显示未匹配？

说明主表中的 `医保属地` 没有在医保属性映射表中找到对应关系。请检查医保属性映射表的医保属地和属性字段是否完整。

### 5.4 可以上传 `~$` 开头的 Excel 文件吗？

不建议上传。`~$` 开头的文件通常是 Excel 打开时生成的临时锁文件，不是完整业务数据。请关闭 Excel 后，选择正式的 `.xlsx` 文件上传。

### 5.5 文件下载到哪里了？

导出后，文件会保存到浏览器默认下载目录。通常可以在浏览器右上角的下载记录中查看，也可以到电脑的 `下载` 文件夹中查找。

### 5.6 没有网络可以使用吗？

可以。工具箱是本地 HTML 页面，双击 `index.html` 即可使用。只要浏览器能打开页面，就可以完成上传、预览、匹配和导出。
